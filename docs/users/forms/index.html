    Supported Forms

    Schoork currently supports the following digital forms for users.

    1. Activity Request - for taking students off campus
    2. Board Item - for submitting items for school board approval
    3. Budget Revision - for revising budgeted amounts for various line items
    4. Bus Request - for requesting bus transportation
    5. Deposit - for tracking the deposit of funds to the bank
    6. Facility Use - for using a district facility after-hours
    7. Fixed Asset Request - for adding, transferring, reporting lost or stolen, and disposing of fixed assets
    8. Fundraiser - for requesting approval to raise funds
    9. Overtime Request - for requesting approval for hourly workers to work beyond forty hours in a week
    10. Personal Activity Report - for reporting time and effort of employees paid partially through federal funds
    11. Requisition - for requesting the purchase of goods and services
    12. Semi-Annual Certification - for reporting time and effort of employees paid wholly through federal funds and one cost objective
    13. Time Off Request - to request short or long-term leave from work
    14. Travel Reimbursement - for requesting reimbursement of travel expenses
    15. Travel Request - for requesting approval to travel and receive advance payment and/or reimbursement of travel expenses

    Initial Setup

    When a user is assigned as a form approver of a certain type, they will recieve an email notification whenever a form of that type is submitted. If a user is selected to be an approver, they also must approve all forms of that type when they are submitted.

    Users with the Admin permission can add and edit form approvers.

    All forms can have users that approve that form type.

    Users can be named as supervisors of other users. When a user submits a form of the types below, that users supervisors are notified of the form, and, unless the supervisor does not have the Approves Forms role, must approve it.

    Users with the Admin permission can add and edit supervisors of other users. For more information about supervisors, visit the Users page.

    The following form types require approval by a user’s supervisors when submitted.

    1. Personal Activity Report
    2. Semi-Annual Certification
    3. Time Off Request
    4. Travel Request

    Below is a listing other setup modules that must be added and the form types for which they are used. Users with the Admin permission can add these modules in the Setup section of the sidebar.

    Some of these modules are documented elsewhere on this site. If they are, that page is hyperlinked to their name below.

    Module Form Types
    Purchasing Code Budget Revision, Requisition, Travel Reimbursement, Travel Request
    Reservation Location Facility Use
    Revenue Source Deposit, Fundraiser
    School Activity Request, Deposit, Fixed Asset Request, Fundraiser, Overtime Request
    Vendor Requisition

    Status of Forms

    The following are the different status levels of forms and what can be done to the form when it is at that status level.

    Status Level User Can Set Editable Approvable Printable
    Saved Yes Yes No No
    Requested Yes No Yes No
    Approved No No No Yes
    Denied No No No No

    The user who submits the form can change the status from Saved to Requested when editing the form. Once a form is set to Requested it’s status can only be changed by those users who have the right to approve or deny it.

    The chart below shows how the status of a form might change over time.

    stateDiagram Saved-->Requested Requested-->Saved Requested-->Denied Requested-->Approved

    Approval

    When the user sets the form’s status to Requested, it now must be approved or denied by one or more users to change status. Each of these approvers has the following three options.

    Option Status Change Reason Required Available via Email Link
    Approve Maybe* - Approved No Yes
    Deny Yes - Denied Yes Yes
    Return for Changes Yes - Saved Yes No

    *If all other approvers have approved form

    Signatures

    In order to approve or deny a form, a user has to set their signature. This is a one-time process and is described in more detail here.

    Special Statuses

    The following forms have special statuses as an intermediary step between Requested and Approved status. When these forms are fully approved, they will move to the intermediary step, needing more information, before they are moved to the Approved status.

    Form Type Intermediary Step Information Needed
    Bus Request Drivers Needed The names of the drivers assigned to the trip
    Deposit Bank Info Needed Bank transaction number and date
    Requisition PO Needed Purchase order number assigned to the requisition

    Files

    All forms support file attachments for documentation purposes. For things like procurement packets, Schoork can serve as an easy way to have the requistion, purchase order, and all necessary documents in the same place.

    Removal of Forms

    Users can remove the forms they submit when they have a status of Saved. This completely removes the form and any associated items with it.

    Users with the Admin permission can remove forms as long as the form’s status is not Approved.

    Removal of forms cannot be undone.
